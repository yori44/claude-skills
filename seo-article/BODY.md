# BODY.md — 本文生成

## STEP 7: 本文生成

承認済みの骨子をもとに、H2セクションごとに順番に本文を生成する。

### 7-1. 生成方針

**全体方針**
- 承認済みの見出し構成を厳守する（見出しは変更しない）
- レギュレーションの文体を全セクションで統一する
- 目標文字数の範囲内に収める
- 禁止語を使用しない

**各セクションの執筆方針**
- 冒頭（リード文）: KWを最初の100文字以内に自然に含める。読者の悩みに共感してから解決策を示す流れで書く
- 各H2セクション: 骨子のE-E-A-T補強ポイントの指示に従い、データ・具体例・注意事項を盛り込む
- H3セクション: H2の内容を具体的に掘り下げる。箇条書きと文章を使い分けて読みやすくする
- FAQ: Q&A形式を守り、簡潔に答える。各回答は100〜200字程度
- まとめ: 記事全体のポイントを3〜5点で整理し、次のアクション（CTA）を含める

### 7-2. 内部リンクの挿入

骨子の内部リンク挿入指示に従い、該当セクションに自然な文脈でリンクを挿入する。

形式：`[アンカーテキスト]({内部リンクURL})`

アンカーテキストのルール：
- リンク先のテーマに合ったKWを含む
- 「こちら」「詳しくはこちら」等の汎用テキストは使用しない

### 7-3. 外部リンクの挿入

E-E-A-T補強ポイントに「外部リンク」の指示がある箇所には、信頼性の高い情報源へのリンクを挿入する。

対象となる情報源の優先順位：
1. 政府機関・公的機関（.go.jp / .gov等）
2. 学術機関・研究機関
3. 業界団体・専門機関
4. 大手メディア・新聞社

### 7-4. セクションごとの進行

各H2セクションを生成するたびに進捗を表示する：

```
H2「{見出し}」を生成しました。（{n}/{total}セクション完了）
```

全セクション完了後に合計文字数を表示する：

```
本文生成完了。合計文字数: 約{n}字（目標: {min}〜{max}字）
```

目標文字数を超過または不足している場合は自動調整を試みる。調整後も範囲外の場合はユーザーに報告する。
