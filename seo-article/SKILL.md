# seo-article

受託SEO記事制作の全工程を一つのワークフローで完結させるスキル。骨子確認を挟み、本文まで生成してチャット上に出力する。

---

## 呼び出し方

```
/seo-article
```

引数なしで呼び出す。スキル起動後にインプットファイルのパスを順番に確認する。

---

## インプット一覧

スキル起動時に以下を確認する。未提供のものがあれば警告してユーザーに入力を求める。

| # | インプット | 内容 | 必須 |
|---|-----------|------|------|
| 1 | キーワードファイル | 1行1KW形式のテキストファイル | 必須 |
| 2 | 目標文字数 | 例：3000〜5000字 | 必須 |
| 3 | レギュレーションファイル | 禁止語・文体・見出しルール等を記載したmdファイル | 必須 |
| 4 | ターゲット読者情報 | 年齢・悩み・ペルソナ等（テキストまたはファイル） | 必須 |
| 5 | 内部リンク先リスト | 内部リンク挿入候補のURL一覧 | 必須 |
| 6 | 参考記事・既存コンテンツ | クライアントの既存記事URL or ファイル | 任意 |

---

## ワークフロー全体

以下の10ステップを順番に実行する。STEP 6でユーザー確認を待つ。

```
STEP 1:  インプット解析・バリデーション
STEP 2:  検索意図分析
STEP 3:  競合URL収集（自動検索）→ 競合分析
STEP 4:  骨子生成
STEP 5:  骨子レギュレーションチェック
STEP 6:  骨子をチャットに出力 ← 【停止・確認待ち】
STEP 7:  本文生成
STEP 8:  ファクトチェック（自動検索 → ユーザー最終確認）
STEP 9:  本文レギュレーションチェック
STEP 10: 完成サマリーをチャットに表示
```

詳細は各サブドキュメントを参照：
- 競合調査・検索意図分析 → [RESEARCH.md](RESEARCH.md)
- 骨子生成 → [DRAFT.md](DRAFT.md)
- ファクトチェック → [FACTCHECK.md](FACTCHECK.md)
- 本文生成 → [BODY.md](BODY.md)
- レギュレーションチェック → [REGCHECK.md](REGCHECK.md)

---

## STEP 1: インプット解析・バリデーション

### 1-1. ファイル読み込み

以下を読み込み、構造化して保持する。

**キーワードファイル**
- 1行1KWとして配列に格納
- KW数を表示する

**レギュレーションファイル**
以下の項目を抽出する：
- 文字数（最小・最大）
- 見出しルール（H2数・H3数の上限・下限）
- 禁止語リスト
- 文体（です・ます調 / だ・である調 等）
- リンクルール（内部リンク数の上限・外部リンクルール）
- その他クライアント固有ルール

**ターゲット読者情報**
- ペルソナ・悩み・ゴールを構造化して保持

**内部リンク先リスト**
- URL一覧を配列に格納

**参考記事**（提供された場合）
- URLの場合はWebFetchで取得
- ファイルの場合はそのまま読み込む

### 1-2. バリデーション

| チェック項目 | 条件 |
|------------|------|
| KWが1件以上あるか | なければエラーで停止 |
| 文字数が指定されているか | なければユーザーに確認 |
| レギュレーションに文体・禁止語が含まれるか | なければ警告して続行 |
| 内部リンク先リストが1件以上あるか | なければ警告して続行 |

バリデーション完了後、以下を表示する：

```
【インプット確認】
KW数: {n}件
目標文字数: {min}〜{max}字
文体: {文体}
禁止語: {n}件
内部リンク候補: {n}件
参考記事: {あり/なし}

処理を開始します。
```

## STEP 6: 骨子をチャットに出力【停止点】

DRAFT.md の OUTPUT形式で骨子をチャットに表示し、以下を表示してユーザーの確認を待つ：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
骨子を出力しました。

内容を確認して、以下のいずれかを入力してください：
  「OK」     → このまま本文生成に進む
  「修正: ○○」→ 修正内容を伝えてください
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

ユーザーが「OK」と入力するまでSTEP 7には進まない。
修正依頼があれば骨子を修正し、再表示してから再確認を求める。

## 複数KWの処理

キーワードファイルに複数KWがある場合、1KWずつ順番に処理する。

処理開始前に確認を表示する：

```
{n}件のキーワードを検出しました。
順番に処理します：
1. {keyword1}
2. {keyword2}
...

処理を開始しますか？（yes / no）
```

各KWの処理が完了するたびに「次のKWに進みますか？」と確認してから次へ進む。
