STEP 2〜3 の詳細手順。

## STEP 2: 検索意図分析

各KWに対して以下を分析する。Claudeの知識を使って推論する。

| 分析項目 | 内容 |
|---------|------|
| 意図分類 | 情報収集型 / 比較検討型 / 購買型 / ナビゲーション型 |
| ターゲット読者像 | どんな人が・何のために・何を知りたいか（インプットのターゲット情報と照合） |
| 適切なコンテンツ形式 | How-to / ランキング / 比較表 / Q&A / 体験談 など |
| 想定する読者の悩み | 記事を読む前の状態・読んだ後に得られるもの |

分析結果を以下の形式でまとめる：

```
【検索意図分析: {keyword}】
意図分類: {分類}
ターゲット: {1〜2文}
コンテンツ形式: {形式}
読者の悩み: {悩み}
読後のゴール: {ゴール}
```

## STEP 3: 競合URL収集・分析

### 3-1. URLの収集

以下のメッセージをユーザーに表示する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【競合URL収集: {keyword}】

Googleで「{keyword}」を検索して、上位5〜10件のURLを貼り付けてください。
（スキップする場合は「スキップ」と入力してください）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 3-2. URL分析（URLが提供された場合）

WebFetchで各URLにアクセスし、以下を抽出する：

| 抽出項目 | 内容 |
|---------|------|
| ページタイトル | title要素またはH1 |
| H2・H3見出し一覧 | 全見出しをリストアップ |
| 推定文字数 | 本文の長さから概算 |
| メタディスクリプション | あれば取得 |

### 3-3. 競合分析まとめ

取得した情報から以下を分析する：

**共通トピック**
- 上位記事の70%以上に含まれる見出し・テーマを必須トピックとして抽出

**平均構成**
- 上位記事の平均H2数・H3数・文字数を算出

**差別化ポイント**
- 上位記事に不足しているが検索意図に合う要素を2〜3個抽出
- ターゲット読者情報・参考記事と照合して独自の切り口を特定

**競合分析サマリー出力形式：**

```
【競合分析サマリー: {keyword}】
参考URL数: {n}件
上位記事平均: H2 {n}個 / H3 {n}個 / 約{n}字

共通トピック（必須）:
- {トピック1}
- {トピック2}
- ...

差別化ポイント（独自要素）:
- {ポイント1}
- {ポイント2}
```

### 3-4. スキップされた場合

競合URLなしでSTEP 4へ進む。骨子備考欄に「競合調査未実施」と記載する。
