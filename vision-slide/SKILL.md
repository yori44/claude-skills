# vision-slide

Visionスクールの講義スライドを生成するワークフロー。  
スライド素材（テキスト）とCodex実装指示書（デザイン仕様＋image2プロンプト）を1セットで作成する。

---

## 呼び出し方

```
/vision-slide
```

引数なしで呼び出すと、以下の情報をステップで確認する。

---

## Step 1：基本情報を確認する

以下を確認する。情報が不足していたら必ず聞く。

1. **章番号・タイトル**（例：第2章② SKILLsとワークフロー）
2. **素材ファイルのパス** — 事前に作成済みのmdファイルがあればパスを教えてもらいReadする。なければチャットに素材を貼り付けてもらう。
3. **必ず1つ以上質問する** — 以下のどれかを聞く：
   - 講師の一次情報・体験談（「この内容に関連する自分の実体験はありますか？」）
   - 強調したいポイント（「特に初心者に伝えたいことは何ですか？」）
   - スライド枚数の想定（「ボリューム感の目安はありますか？」）

---

## Step 2：スライド構成を設計する

素材の内容量に応じてスライド枚数を**自動調整**する（10枚固定ではない）。  
目安：

| 内容量 | 枚数目安 |
|--------|---------|
| 概念1〜2個 | 6〜8枚 |
| 概念3〜4個 | 10〜12枚 |
| 概念5個以上 | 13〜16枚 |

必ず含めるスライド：
- 表紙（タイトル＋サブタイトル）
- アジェンダ（この章で学ぶこと）
- まとめ（ポイントの箇条書き＋次章への橋渡し）

---

## Step 3：スライド素材mdを生成する

**保存先**: `c:/Users/yokan/Desktop/MYHOME/04_プロジェクト/Vision/` 配下に適切なフォルダを作成して保存。

**テキストルール**:
- 講師名・個人名は出さない
- ですます・体言止めで統一
- 1スライド1メッセージ
- 講師の一次情報は「講師コメント」として記載
- テキストは簡潔に（動画講義を想定し、読み上げ原稿にはしない）

---

## Step 4：Codex指示書mdを生成する

**保存先**: 素材mdと同じフォルダ。

### 全スライド共通のデザイン仕様（必ず埋め込む）

```
### カラーパレット
- 背景: #0D0D0D（ほぼ黒）
- アクセント1（メイン）: #4F8EF7（ブルー）
- アクセント2（強調）: #F7C94F（イエロー）
- テキスト（白）: #FFFFFF
- テキスト（サブ）: #AAAAAA

### フォント
- タイトル: Noto Sans JP Bold / 40〜48pt
- 本文: Noto Sans JP Regular / 18〜22pt
- 補足・キャプション: Noto Sans JP Light / 14pt

### レイアウト原則
- 余白は最小限に抑える。テキストエリアと画像エリアでスライド全体を埋める
- 余白が生じる場合はテキストボックス（補足・ポイント・講師コメント等）を追加して埋める
- 1スライド1メッセージ
- テキストは左寄せ基本、タイトルは左上
- 画像は右半分 or 中央下に配置
- アイコン・図解を積極活用して超初心者でも視覚的に理解できるように

### AIツールロゴの使用（積極的に挿入すること）
- ロゴはローカルパス `C:/Users/yokan/Desktop/MYHOME/04_プロジェクト/Vision/AIロゴ/` から読み込む（python-pptxのadd_picture）
- ファイル名対応: ChatGPT.jpg / Claude.png / Gemini.jpeg / Manus.jpeg / genspark.png / antigravity.jpg / claude code.jpg / obsidian.jpg / openclaw.jpeg / X.jpg / note.webp / Discord.jpg / canva.jpeg / Grok.png / Midjourney.jpeg / Typeless.png
- **挿入ルール（必須）**:
  - ツール名がテキストに登場したら、必ずその横または直上にロゴを配置する
  - 「代表ツール」「使用ツール」「必要なもの」などのリスト項目は全件ロゴ付きにする
  - 比較表・カテゴリ図・フロー図でもツール名が出たら全てロゴを入れる
  - 表紙・アジェンダでも章に関連するツールロゴを右側の装飾として並べる
- **ロゴのサイズと配置**:
  - リスト項目内: 高さ28〜36px、テキストの左横に配置（アイコン→テキストの順）
  - カテゴリボックス内: 高さ40〜48px、ボックス上部中央に配置
  - 比較表のヘッダー行: 高さ32px、ツール名テキストの左横
  - 装飾配置（表紙など）: 高さ48〜64px、半透明（opacity 70%）で右側に並べる
- ロゴが存在しない場合はツール名をブランドカラーのテキストボックスで代替
```

### image2プロンプトのテイスト

- **スタイル**: ホログラム・ネオン・ダークオフィス・デジタルアート風（実写調）
- 人物は出さない or 後ろ姿・シルエットのみ
- 必ず `dark background, blue neon accents, high quality digital art` を末尾に含める
- スライドの内容を象徴するビジュアル（概念図・UI・フロー・ツールアイコン等）

### Codex実装メモ（必ず末尾に追加）

```
## Codex実装メモ

- ライブラリ: python-pptx
- フォント埋め込み: Noto Sans JP（事前にインストール確認）
- 画像生成: image2 API で各スライドの画像を生成してPPTXに挿入
- スライドサイズ: 16:9（33.87cm × 19.05cm）
- 出力ファイル名: vision_chapter[章番号]_[英語タイトル].pptx
- Google Slides変換: PPTXを google-slides-api または slides-converter で .gslides / Google Driveにアップロードして変換。python-pptx生成後に自動変換するスクリプトも追加すること。
- **実行順序（必須）**: このmdファイルを読み込んで動作を開始する前に、まず全スライド分のimage2プロンプトを使ってGPT IMAGE 2で画像を全件生成し、ローカルに保存する。画像生成が完了してからpython-pptxでのスライド組み立てを開始し、生成済み画像をadd_pictureで埋め込むこと。
```
