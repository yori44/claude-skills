# AI副業漫画 制作ワークフロー (/ai-manga)

タクト&サヤのAI副業漫画を半自動で制作する。漫画は「作品」ではなく**LINE集客のストック資産**として作る（[[ai_manga_project]]）。

- **入力**: テーマ（未指定ならStep1で候補出し）
- **出力**: 本文8000〜10000字 ＋ 漫画CSV（46枚）＋ codex画像生成コマンド ＋ Docs組版テキスト

**構造＝各ブロックが「見出し → 漫画N枚（連続）→ テキスト」の繰り返し**
```
はじめに見出し → 漫画8枚 → はじめにテキスト
第1章見出し   → 漫画6枚 → 第1章テキスト
第2章見出し   → 漫画6枚 → 第2章テキスト
…（第5章まで）
おわりに見出し → 漫画8枚 → おわりにテキスト
```
**漫画枚数（確定）：はじめに8＋第1〜5章×6＋おわりに8＝合計46枚**（2月版45枚と同規模）。

**最重要の型＝「漫画＝問題提起／本文＝答え」（フック→回収）**
各ブロックの漫画群は謎・不安・絶望で止める（答えを漫画のセリフで言わない）。直後のテキストがその漫画を受けて回収・解説する。2月版の失敗＝本文をそのまま漫画化して重複した。これを根絶する。

**自動化の境界（確定設計）**
| 工程 | 自動化レベル |
|---|---|
| ① ネタ探し | 半自動（候補出し→ユーザーが選ぶ） |
| ② KW抽出 | 全自動 |
| ③ 本文8000字 | 全自動 |
| ④ 漫画CSV | 全自動（ユーザーがチェック） |
| ⑤ 画像生成 | 半自動（codexコマンド46本を生成→ユーザーが叩く） |
| ⑥ 組版 | 半自動（プレースホルダ自動→ユーザーが画像手ドラッグ） |

## Step 4: ④ 漫画シナリオ設計＋CSV化（全自動・ユーザーチェック）

本文から「どこをフックにするか」を Claude が判断 → コマ割りテンプレ選択 → セリフ → 英語プロンプト → CSV出力。**漫画＝計46枚**（はじめに8／各章6／おわりに8）。

**コマ割りテンプレ（出典 `08_コンテンツ制作/マンガ生成フロー.md`）**
| テンプレ | レイアウト | 使いどころ |
|---|---|---|
| 1 | 1コマ | 章扉・見せゴマ・締め |
| 2 | 2コマ均等 | 対比・会話 |
| 3 | 2コマ上小下大 | 問い→回答・疑問→気づき |
| 4 | 2コマ上大下小 | 状況→コメント |
| 5 | 3コマ縦並び | ステップ・3つの特徴 |
| 6 | 3コマ上1下2 | 概要→2者の反応 |
| 7 | 3コマ上2下1 | 会話→結論 |
| 8 | 4コマ起承転結 | 4ステップ・Q&A |
| 9 | 4コマ右大モックアップ | 操作画面を右大表示 |
| 10 | 4コマ左大モックアップ | 完成画面を左大表示 |

⚠️ **codexはテンプレを厳密に守らない**（テンプレ6指定→4コマ寄りで出た実績）。構図は「だいたいこの配置」程度の指定に留める。
- **横長3コマ縦積みを主力に**（真ん中はセリフだけ等、情報量を抑えて緩急）。基本3〜5コマ。テンプレ選択はこれを軸に簡素化。

**🔴🔴 読了率の設計（KU印税直結・石川先生／[[★最重要_漫画読了率の設計論_石川先生]]）**
KU印税＝読了率（既読ページ数）。ここが最優先。CSV生成時に必ず組み込む：
- **引き・めくり**：各漫画ブロック末に「引き（疑問・動作の初動・新事実を1つ）」を作り、直後の本文で回収。漫画ページ末の引きで本文へ引き込む＝KU既読ページUP。
- **振り（予兆）**：AIは振りを飛ばす（最短距離でオチにいく）ので、感情を乗せたいシーンには人力で振り（予兆・前フリ）を追加。シーン転換にワンカット挟む。
- **顔漫画回避**：4コマ以上のページで全コマに等身大キャラを乗せない。背景だけの「気休めコマ」を挟む／全員出すならちびキャラ混在（⑤の目滑り対策と連動）。

**🔴🔴🔴 因果カット禁止＝コマ間の飛躍をなくす（7/2実走で最大の失敗）**
AIは放っておくと**シーンを最短距離でダイジェスト化**する。「1ページ＝1シーン」で8シーン詰めると、コマとコマの間の"なぜそうなった"が全部飛んで、読者が「え、もう解決？意味わからん」になる（実際7/2に食らった）。防ぐルール：
- **1ページに新シーンを詰め込まない**。1つの出来事（例：ノウハウ入手→実行→成果）は**因果を1コマずつ刻む**。ページ数は惜しまず増やす（はじめにでも8→12ページに増やして正解だった）。
- **登場・変化には必ず「理由のコマ」を置く**。新キャラがヌッと出る／状況が急変するのは厳禁。7/2例＝サヤ登場が唐突→「タクトが作ったAI美女(サヤ)が画面から喋り出す」の理由コマを足して自然になった。
- **CSV確定前に「このコマとコマの間、読者は流れを追えるか？」を1ページずつ自己チェック**。追えないなら間のコマを足す。これは品質ゲートの必須項目。
- **専門用語・界隈用語を漫画に入れない**（初心者に通じない）。7/2例＝「驚き屋」「順番」等の曖昧語・内輪語はNG。誰でも分かる具体語にする。
- **トーンは暗くしすぎない**。ヘタレ主人公の自虐・ズッコケでコミカルを混ぜて緩急を出す（絶望一辺倒は読んでてしんどい）。
- **セリフは最後に人間が手直し前提**（AIのセリフは硬い。石川式）。

**🔴🔴 漫画の基本型＝「タクトが詰まる → サヤが教える → 本編はこっち！」（毎話これで組む）**
どの章・どの話でも、漫画パートの骨格はこの3ステップで固定する（7/3ふりー指示で確定）：
1. **タクトがAI・副業の実務で困る**（美女しか作れん／稼ぎ方が分からん／ツールが使いこなせん等、その章のテーマの"詰まり"を見せる）
2. **サヤが教えてあげる**（クール姉貴が答えの入口・気づきを与える。※漫画では答えを言い切らず、方向性だけ示す＝答えの本体は本文）
3. **「本編（本文）はこっちよ！」の引き**（サヤが本編へ誘導。「続きは本編。ついといで。」等。読者を本文＝KU既読ページへ流す）
→ 漫画＝問題提起＋誘導、本文＝答え、の役割分担を毎話この型で徹底。ラスト（おわりに章）だけLINE誘導に切り替える。

**中間工程＋品質ゲート（中島ゆか式・AIC）**
1話（1章）ずつ生成し、キャラ名を具体化、コマ数を明記、**「この設定・展開に不自然な点は？」と自己チェック**してからCSV確定。ストーリーの型＝「現状の悩み→解決のきっかけ→解決案→良かったね→(次の悩み)でループ」を1章1ループで納める。全体にヒーローズジャーニー（日常→事件→試練→変化→帰還）を薄く敷く。

**セリフ設計**
- 1コマ1〜2文。タクトが疑問/絶望（漫画＝問題提起、答えは言わない）→ ※答えは本文で回収。
- タクト＝ヘタレ敬語、サヤ＝クール姉貴口調。
- **1吹き出し25文字・句読点なし・複数フォント（やさしさアンチック等）**。カラー漫画は1ページ3コマ基準。
- P8（おわりに）にLINE QR用の空き枠を確保。

**CSV列定義**（既存 `セリフ有り参考.csv` と互換）
```
ページ番号,使用するコマ割りテンプレ,漫画作成のプロンプト
1,テンプレ3,"Generate a SINGLE complete manga page..."
```

## Step 5: ⑤ 画像生成 — codexで自動生成（無承認・BG実行）

**codex実行は無承認で自動化する**（`codex exec` は settings.json のアローリスト済み＝承認プロンプト無し）。SKILLがコマンドを生成し、**そのままバックグラウンドで自動実行する**（ユーザーが手で叩く必要なし）。Bashツールのタイムアウト10分制約があるので**1枚1コマンドで、章単位のまとまりをBG実行→完了通知を待つ**。

⚠️ **事前準備**：タイトルフォルダを作り、`タクト.png`・`サヤ.png` を作業ルート（`Desktop\漫画\`）からそこへコピーしてから叩く（混線罠②回避）。生成中はファイル移動しない。

各ページのCSVプロンプトから以下フォーマットのコマンドを生成し、cwd＝タイトルフォルダで自動実行：

```bash
cd "/c/Users/yokan/Desktop/漫画/[漫画タイトル]"
ls -la タクト.png サヤ.png   # ← 混線の罠①対策。実在確認してから生成（タイトルフォルダにコピー済みのもの）
codex exec --skip-git-repo-check "{英語プロンプト＋日本語セリフ}" -i "タクト.png" -i "サヤ.png"
```

**プロンプトのコツ**
- 冒頭に `Generate a SINGLE complete manga page (ONE image, vertical 3:4)` ／ `Do NOT output separate images`。
- 英語で書く。**セリフは日本語のまま**書く（文字化けしない）。
- キャラ一貫を最優先で明記。`-i` は1枚目＝主役（タクト）を先に。
- **`Japanese text only, no English UI text`** を最初から入れる（背景の英語UI混入を防ぐ）。
- 描かないキャラの特徴を否定形で打ち消す（例 `NOT Free, NOT another man`）。
- **目滑り対策**：全コマにキャラを出さない。2ページに最低1コマ「キャラが主役でないコマ（背景・小物・空・手元・セリフだけ）」を入れる。プロンプトを「人物◯◯」固定にしない（④の顔漫画回避と連動＝AI漫画っぽさ撲滅）。
- **アングルを散らす（単調にしない）**：`from side`／`from behind`／`oblique`／`from above`（俯瞰）／`low angle`（煽り）／`over the shoulder`（背中越し）／`close-up on feet/hands` を各ページで散らす。同じ正面バストアップの連続を避ける。

**実測**：1枚約2〜4分・トークン約2〜4万。46枚なら章ごとに分割して回す（全部で2〜3時間規模＝1日仕事）。**バックグラウンド実行＋1枚ずつ**。事故った枚だけ止めて打ち直す。

**保存先**：codexは `~/.codex/generated_images/{session_id}/ig_*.png` に吐く（1 exec=1フォルダ=1画像）。隠しフォルダなので、**生成後すぐタイトルフォルダ（`Desktop\漫画\[タイトル]\`）へ `P01.png`〜 の連番でコピー**する。

### Step 5.5: ⑤b サイズ自動補正（全ページ 1600×2560 で固定）

⚠️ gpt-image-1は出力解像度が固定値（1024×1536等）に丸められ、ジャストサイズでは出ない。3:4前後の縦長で出る。→ **生成後に必ず全ページを 1600×2560 へ統一する**（表紙もこのサイズ＝縦横比5:8で本文と揃う・KDP準拠）。最終的な横幅はDocs上で調整するので、**出力段では例外なく1600×2560固定**にしておくこと。

**方式＝Pillowでパディング（切れない）**。アスペクト維持で1600×2560に収め、余白は白で埋める。端が切れると絵が欠けるのでクロップは使わない。生成スクリプト（`gen_manga.py`）に組み込み済みで、全ページ生成後に一括でかける：

```python
from PIL import Image
def fit_pad(path, size=(1600, 2560), bg=(255, 255, 255)):
    im = Image.open(path).convert("RGB")
    if im.size == size: return
    tw, th = size
    s = min(tw/im.width, th/im.height)
    nw, nh = round(im.width*s), round(im.height*s)
    im = im.resize((nw, nh), Image.LANCZOS)
    c = Image.new("RGB", size, bg); c.paste(im, ((tw-nw)//2, (th-nh)//2)); c.save(path)
```

これで「デフォでサイズ適合した画像」が出る。Docs/Kindle組版時の縮み問題（A4本文幅約16cm超で自動圧縮される件）は組版側の幅設定で対応：**画像は「行内」配置＋幅16cm＋ページ余白を狭める**。

## Step 6: ⑥ Docs組版（半自動）

⚠️ **Docs MCPは画像インライン埋め込み不可**。→ 半自動が現実解：

1. **Google Docs MCP（create_file）でテキスト本文を自動生成**
2. 各ブロック頭に **`【ここに漫画◯を挿入】`** プレースホルダを自動配置
3. **ユーザーが漫画画像を手でドラッグ挿入**（行内・幅16cm）
4. Kindle最終組版・EPUB化・KDP入稿は手動 or 既存ルート

**🔴 KDP入稿の地雷（収益を守る最重要）**
- **出版区分は必ず「電子書籍」で出す**。「マンガ／コミック」区分を選ぶと**Kindle Unlimited対象外＝KU印税ゼロ**。ハイブリッド本（漫画フック＋本文）は「電子書籍」が正解。フル漫画にせずハイブリッドにしているのはこのため。
- **表紙はJPEG必須・1600×2560px**（PNG/PDFは出版不可で弾かれる）。

**本文側の読了率対策（漫画側＝石川先生と両面で離脱を下げる）**
- Kindle Previewerでスマホ表示チェック（改行・図の縮み）／リンク付き目次の動作確認／図解は1ページ取り／本文は3〜4行で改行（スマホで文字塊にしない）。

将来：完全自動化したい場合はHTML/EPUB経路を別途検討。
